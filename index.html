Identidad

Debemos configurar nuestra identidad en git

Si no la primera vez que utilizemos la herramienta

Nos pedira la misma.


                        $ git config --global user.name "John Doe"
                        $ git config --global user.email johndoe@example.com

Parametros configurados

Para saber los parámetros que tenemos configurados

En el terminal escribimos los siguiente


                            $ git config --list

Help

Para obtener ayuda de los comandos


                            $ git help
                            $ git help |verb|

Blobs

Representan el contenido de uno o varios archivos

Se identifica por su SHA1 no por su nombre

El único que sabe a que archivo corresponde cada blob es el TREE

No hay copias de los blobs.

Tree

El objeto tree relaciona los blobs con los nombres de los ficheros.

Commit

El commit es un snapshot del estado de los ficheros en un momento concreto

No se almacenan deltas como en SVN sino ficheros enteros.

No hay Blobs duplicados.

Se identifican por su SHA1

Commit

Tag

Son etiquetas que se pueden aplicar a commits.

Por ejemplo para identificar versiones estables de un producto.

Head

Es un puntero que apunta al commit actual

Remote

Es la url de una copia del repositorio

Origin

Es el remote desde el que se clonó el repositorio inicalmente

Branches

Una rama es una lista de commits

Se suele representar como un puntero a un commit

Cuando se hace commit, el HEAD de la rama se mueve al nuevo commit que ahora tiene como padre el commit anterior

Index

Es un fichero binario(cache)

Dinámico y temporal

Representa la estructura de directorios actual del repo

Captura la versión del repo completo en algún momento del tiempo

Index

El contenido del índice es que se metera en el próximo commit
